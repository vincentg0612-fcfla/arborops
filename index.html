Overview

What we built

ArborOps

A custom operations platform for employees, managers, foremen, mechanics, estimators, and crews. It powers jobs, estimates, routing, scheduling, portal access, and workforce workflows.

Olive

A conversational internal assistant embedded into the iPhone app and employee portal. Olive helps people get answers, make decisions, and perform real work instead of just chatting.

Why it matters

Most field-service operations tools make people bounce between scheduling screens, weather sites, notes, route apps, and team messages. ArborOps + Olive turns that into a single workflow. Employees can ask natural questions like “What do I need to know today?”, “What still needs to be scheduled?”, “What’s the forecast in Clermont tomorrow morning?”, or “Schedule that follow-up for Monday at 9.”

Features

What Olive can do

Role-aware operations assistant

Olive uses ArborOps context to answer differently for managers, schedulers, foremen, mechanics, estimators, and crews.

Today summary

Understands today’s work, calendar items, key announcements, and operational priorities.

Scheduling gap detection

Surfaces unscheduled jobs, pending estimates, and missing crew assignments.

Schedule recommendations

Recommends what to schedule next based on backlog, distance clustering, current work, and route relevance.

Schedule execution

After confirmation, Olive can schedule jobs or estimates through a live Odoo endpoint.

Weather tied to work

Checks weather for today’s jobs or estimates using context-aware locations and Open-Meteo data.

Weather anywhere

Understands direct place-based questions such as Clermont tomorrow morning or Orlando this afternoon.

Route recommendations

Uses geocoding and routing logic to suggest better stop order and travel planning.

Reminder and follow-up tasks

Creates live reminder tasks in ArborOps so Olive can behave like a personal secretary, not just an assistant.

Voice conversation

Integrated with Azure Speech for microphone input and spoken responses using Ava Dragon HD voice.

iPhone app integration

Employees can talk to Olive directly inside the ArborOps mobile app with Azure-authenticated access.

Portal integration

Olive is also available inside the desktop employee portal so workers can use her from the office or a laptop.

Architecture

How it works

Clients

  • ArborOps iPhone app
  • ArborOps web portal

Identity

  • Azure authentication with MSAL
  • Dedicated Olive API audience
  • Validated bearer tokens in n8n

AI orchestration

  • n8n workflow engine
  • Azure AI for chat and reasoning
  • Azure Speech for voice

Operational data

  • Odoo / ArborOps context APIs
  • Scheduling, estimates, jobs, crews
  • Reminders and follow-up tasks

External tools

  • Open-Meteo for weather + geocoding
  • OpenRouteService for routing + geocoding

Design principle

Olive is not a generic chatbot. She is an operational layer on top of ArborOps. The system fetches only the context needed for the user’s intent, then lets AI explain and act on it in a conversational way.

Submission description

ArborOps + Olive is an AI-powered operations system built for real-world field service teams. We created a custom employee platform called ArborOps and layered Olive on top as a conversational internal assistant that works inside both the iPhone app and the employee portal.

Olive is not just a chatbot. She understands role-based operational context from ArborOps, including jobs, estimates, crews, scheduling gaps, maintenance, reminders, and route data. Managers can ask what still needs to be scheduled, what the forecast looks like for work sites, what the best route is for today’s estimates and jobs, or what they should schedule next. Olive can also create live reminder and follow-up tasks and, after confirmation, schedule work directly through the backend.

The system uses Azure authentication for the app, Azure AI for reasoning, Azure Speech for voice interaction, n8n for orchestration, Odoo/ArborOps for business logic and data, Open-Meteo for weather, and OpenRouteService for geocoding and routing. We designed Olive to feel like an internal personal secretary for operations, not a generic AI. She answers naturally, supports voice and text, and helps employees act faster without jumping between multiple tools.

This project showcases how OpenAI Codex helped us design, debug, and ship a full-stack internal assistant across authentication, workflow orchestration, iPhone integration, employee portal integration, scheduling logic, route planning, reminders, and operational UX. The result is a practical AI system built for a real business workflow with measurable daily value.

Built with OpenAI Codex, Azure AI, Azure Speech, n8n, Odoo, Open-Meteo, and OpenRouteService.
